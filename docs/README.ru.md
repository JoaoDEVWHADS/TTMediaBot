# TTMediaBot

**Привет! Я Жуан Алмейда (João Almeida).** Добро пожаловать в мой форк **TTMediaBot** — многофункционального бота для трансляции музыки и медиафайлов на серверах TeamTalk 5. Этот репозиторий ориентирован на постоянные улучшения, стабильность и добавление новых функций, таких как эксклюзивная поддержка YouTube Music.

> 🔗 **Мой репозиторий:** [https://github.com/JoaoDEVWHADS/TTMediaBot](https://github.com/JoaoDEVWHADS/TTMediaBot)

---

Бот для трансляции медиа на серверах TeamTalk 5 с поддержкой воспроизведения музыки из различных источников (YouTube, YouTube Music, локальные файлы, прямые ссылки) и расширенными функциями управления.

## 🎵 Поддержка YouTube Music

Этот форк предлагает оптимизированную интеграцию с **YouTube Music** наряду со стандартным поиском YouTube:

- **Поиск через YouTube Search API:** Быстрый и надежный поиск треков без задержек.
- **Оптимизированные библиотеки:**
  - Поиск YouTube использует `py-yt-search` — легковесную и быструю библиотеку для Python.
  - Поиск YouTube Music работает на базе `ytmusicapi` — официальной библиотеки API YouTube Music.
  - Скачивание и декодирование аудио в обоих сервисах выполняет `yt-dlp`.
- **Фокус на производительности:** Минимизация задержек при подключении и поиске, плавный запуск воспроизведения.
- **Единая система авторизации:** Сервисы YouTube и YouTube Music используют один общий файл cookies для обхода ограничений платформы.
- **📦 Загрузка плейлистов и альбомов:** Полная поддержка скачивания коллекций через команду `dlp` с автоматической упаковкой в ZIP-архив и сохранением оригинальных тегов.
- **🕵️ Отчет о прогрессе в ЛС:** Бот отправляет сообщения о ходе загрузки плейлиста в приватном чате, чтобы не засорять текстовый канал TeamTalk.

Переключение между сервисами поиска выполняется командой `sv`:
- `sv yt` — Переключиться на YouTube
- `sv ytm` — Переключиться на YouTube Music

> [!NOTE]
> **Эксклюзивная функция:** Поддержка YouTube Music доступна только в этом форке и отсутствует в оригинальном репозитории TTMediaBot.

## 🔗 Загрузка по ссылкам и локальное хранилище

Бот оснащен продвинутой системой загрузки медиафайлов по прямым ссылкам. Вы можете добавлять ссылки в список, просматривать их и скачивать архивом или по отдельности.

### Команды списка загрузок
- **`aad [ссылка]`**: Добавить одну ссылку в ваш список загрузок.
- **`ad [ссылка1] [ссылка2] ...`**: Добавить несколько ссылок через пробел.
- **`ld`**: Показать список всех добавленных ссылок.
- **`rd [номер/ссылка]`**: Удалить ссылку из списка по ее порядковому номеру или тексту URL.
- **`ldd [ссылка]`**: Скачать файл по ссылке напрямую и сразу загрузить в канал TeamTalk.
- **`ads`**: Начать загрузку всего вашего списка. Бот предложит выбрать:
  - **Вариант 1 (Обычный):** Скачать файлы по очереди и отправить каждый в канал.
  - **Вариант 2 (ZIP):** Скачать все треки, упаковать их в один архив ZIP и отправить в канал.
- **`adsc`**: Переключить **режим локального сохранения** на сервере (сбрасывается при перезапуске):
  - При включении файлы из списка `ads` сохраняются прямо на диске VPS, а не отправляются в TeamTalk.
  - Файлы Варианта 1 сохраняются в папку `data/Downloads/music/` (на хосте: `bots/имя_бота/Downloads/music/`).
  - Файлы Варианта 2 (ZIP) сохраняются в `data/Downloads/zips/` (на хосте: `bots/имя_бота/Downloads/zips/`).
  - Локально сохраненные файлы не подлежат автоудалению.

## 🚀 Простая установка (Рекомендуется)

Скрипт автоматически установит Git (если он отсутствует), склонирует проект и настроит Docker.

1. **Скачайте и запустите установщик:**
   ```bash
   wget https://raw.githubusercontent.com/JoaoDEVWHADS/TTMediaBot/master/install_git_clone.sh
   sudo chmod +x install_git_clone.sh
   sudo ./install_git_clone.sh
   ```

2. **Следите за выводом в терминале:**
   - Установщик настроит Docker, системные библиотеки и зависимости.
   - Скрипт выдаст интерактивное меню, в котором вы сможете создавать ботов, управлять ими и обновлять код.

## 🐳 Панель управления Docker (`ttbotdocker.sh`)

Скрипт `ttbotdocker.sh` предоставляет удобный текстовый интерфейс для развертывания и администрирования ваших ботов.

### Основные возможности меню:

#### 1. Создать бота (Create Bot)
Пошаговый мастер создания бота:
- Настройка имени контейнера и папки конфигурации.
- Выбор сервера, портов (TCP/UDP), включение шифрования.
- Ввод логина и пароля бота на сервере.
- Указание пути к cookies.txt.
- Поддержка создания пула ботов (пакетное создание с автонумерацией).

#### 2. Управление ботами (Manage Bots)
Пункт меню, включающий 12 инструментов администрирования:
- **Запустить всех ботов:** Запуск ранее остановленных контейнеров.
- **Перезапустить всех ботов:** Быстрый перезапуск для применения конфигураций.
- **Остановить всех ботов:** Безопасное завершение работы контейнеров.
- **Удалить бота:** Интерактивное удаление контейнера и папки конфигурации.
- **Пакетное удаление:** Удаление пула выбранных ботов (или всех сразу при вводе `0`).
- **Дублировать бота:** Клонирование существующего бота с созданием новых копий и автонумерацией никнеймов.
- **Обновить cookies (для всех ботов):** Быстрое копирование нового файла `cookies.txt` всем ботам с последующим автоперезапуском.
- **Перезапуск по таймеру:** Остановка ботов с обратным отсчетом перед включением.
- **Массовое изменение настроек:** Смена адреса сервера, портов, шифрования или учетных записей для группы ботов.
- **Резервное копирование:** Создание архивов `.tar.gz` настроек в папке `backups/` для удобного переноса.
- **Очистить логи:** Удаление старых файлов журнала `*.log` для освобождения места на диске.

#### 3. Пересобрать образ / Обновить код (Rebuild Image / Update Code)
Скачивает свежие обновления проекта из GitHub и полностью пересобирает Docker-образ бота без кэширования.

#### 4. Удалить всё (Uninstall Everything)
Безвозвратное удаление всех ботов, конфигураций, контейнеров и скачанных Docker-образов с сервера.

#### 5. Проверить обновления (Check for Updates)
Сравнение локального кода проекта с репозиторием GitHub, создание резервных копий и безопасное обновление.

#### 6. Автообновления
Включение и настройка фоновой службы автообновления через системный сервис systemd.

#### 7. Очистить кэш Docker (Docker Clean)
Освобождение дискового пространства без влияния на запущенных ботов: удаление кэша сборщика, неиспользуемых образов и очистка системного журнала `journalctl`.

## 🍪 Конфигурация Cookies для YouTube и YouTube Music

Использование файлов cookies **критически важно** для обхода блокировок воспроизведения со стороны YouTube.

### Как получить рабочий cookies.txt:

1. Авторизуйтесь под своим аккаунтом Google на сайте `youtube.com` в браузере (Chrome, Edge, Firefox).
2. Установите расширение для экспорта:
   - Chrome/Edge: [Get cookies.txt LOCALLY](https://chrome.google.com/webstore/detail/get-cookiestxt/bgaddhkoddajcdgocldbbfleckgcbcid).
   - Firefox: [cookies.txt](https://addons.mozilla.org/en-US/firefox/addon/cookies-txt/).
3. Находясь на вкладке `youtube.com`, откройте расширение и нажмите **"Export All Cookies"** (или Download), чтобы сохранить файл на свой компьютер.
4. Загрузите файл на ваш сервер VPS (например, в `/root/cookies.txt`).
5. Укажите этот абсолютный путь при создании или обновлении бота в скрипте.

При устаревании авторизации и появлении ошибок воспроизведения просто скачайте новый файл cookies и обновите его для всех ботов через пункт `2.7` в `ttbotdocker.sh`.

## 🔧 Устранение неполадок

### Бот не играет музыку с YouTube
- Проверьте актуальность вашего cookies-файла. Авторизация в Google могла закончиться. Скачайте свежие cookies и обновите их в боте.
- Убедитесь в правильности пути к cookies.txt в файле конфигурации `config.json`.

### Бот не появляется на сервере TeamTalk
- Проверьте адрес сервера, TCP/UDP порты и учетную запись в `config.json`.
- Если сервер защищен SSL-шифрованием, убедитесь, что параметр `"encrypted"` в настройках имеет значение `true`. Бот сам скачает и верифицирует сертификат сервера.

### Бот подключился к каналу, но звука нет
- В редких случаях служба PulseAudio внутри контейнера может зависнуть. Перезапустите контейнер бота:
  ```bash
  docker restart [имя_бота]
  ```
- Проверьте уровень громкости команды `v` (например, отправьте `v 50`).
